---
name: session-logger
description: >
  Triggered by /summarize or /log or /session at the end of a Claude Code session.
  Appends a structured summary entry to session-log.md in the project root,
  capturing context, decisions made, files changed, and next steps.
  Use this to preserve continuity between Claude Code sessions without
  manually copy-pasting context into new conversations.
---

# Session Logger Skill

## Trigger
Activates when the user types `/summarize`, `/log`, or `/session`.

---

## STEP 1 — Collect Session Context

Gather before writing:
- What was the main goal of this session?
- What files were created or modified?
- What decisions were made (and why)?
- What was left incomplete or deferred?
- What should the next session start with?

---

## STEP 2 — Write Entry Directly via Bash

Do NOT rely on any external script. Write the entry directly:

```bash
# 1. Create session-log.md with header if it doesn't exist
[ ! -f session-log.md ] && printf "# Session Log\n\nAuto-generated by session-logger. Do not edit manually.\n\n" > session-log.md

# 2. Append the new entry (Claude fills in the values)
cat >> session-log.md << 'EOF'
---
## Session: [YYYY-MM-DD HH:mm]

**Goal:** [one sentence]

**Completed:**
- [file or task]

**Decisions:**
- [decision + reason]

**Deferred:**
- [what was left + why]

**Next Session Should:**
- [specific actionable starting point]

**Key Files:**
- `path/to/file.ts` — [what changed]

EOF
echo "✅ Session entry appended to session-log.md"
```

---

## STEP 3 — Entry Format (fill every field)

```markdown
---
## Session: 2026-03-24 14:30

**Goal:** Build the product listing page for Lugar Store v2.

**Completed:**
- `src/app/features/products/pages/product-list.page.ts` — created
- `src/app/features/products/services/product-state.service.ts` — added filters signal

**Decisions:**
- Used cursor pagination over offset — product table will grow to millions of rows
- Kept filter state in service not URL params — not required for MVP

**Deferred:**
- Mobile responsive layout — design not ready yet
- Empty state illustration — asset pending from designer

**Next Session Should:**
- Run `npm run build` and fix any type errors first
- Then implement the product detail page

**Key Files:**
- `src/app/features/products/` — new feature folder, all files created fresh
---
```

---

## Rules

- **Never truncate** existing entries in session-log.md
- **Always include file paths** for created or modified files
- **Be specific** in "Next Session Should" — vague entries are useless
- **Keep entries focused** — max ~20 lines per entry
- **Commit session-log.md** alongside code so it stays in sync with git history
