# Session Logger

End-of-session memory. Writes one dated, structured entry to `session-log.md` so
the next session (via `session-restore`) can pick up with full context — no
manual paste required.

## When this fires

The user types `/summarize`, `/log`, or `/session` — or asks to wrap up, capture,
or hand off the session.

## Step 1 — Collect the session context

Gather before writing:

- The main **goal** of the session.
- Files **created or modified**.
- **Decisions** made, and why.
- What was **deferred** or left incomplete.
- What the **next session should start with**.

## Step 2 — Append the entry (inline, no external script)

Write it directly — never rely on a bundled script:

```bash
# Create the log with a header if absent
[ ! -f session-log.md ] && printf "# Session Log\n\nAuto-generated by session-logger. Do not edit manually.\n\n" > session-log.md

# Append the new entry (Claude fills the values)
cat >> session-log.md << 'EOF'
---
## Session: [YYYY-MM-DD HH:mm]

**Goal:** [one sentence]

**Completed:**
- [file or task]

**Decisions:**
- [decision + reason]

**Deferred:**
- [what was left + why]

**Next Session Should:**
- [specific actionable starting point]

**Key Files:**
- `path/to/file.ts` — [what changed]

EOF
echo "✅ Session entry appended to session-log.md"
```

## Step 3 — Fill every field (worked example)

```markdown
---
## Session: 2026-03-24 14:30

**Goal:** Build the product listing page for Lugar Store v2.

**Completed:**
- `src/app/features/products/pages/product-list.page.ts` — created
- `src/app/features/products/services/product-state.service.ts` — added filters signal

**Decisions:**
- Cursor pagination over offset — the product table will grow to millions of rows
- Filter state in the service, not URL params — not required for MVP

**Deferred:**
- Mobile responsive layout — design not ready
- Empty-state illustration — asset pending from designer

**Next Session Should:**
- Run `npm run build` and fix type errors first
- Then implement the product detail page

**Key Files:**
- `src/app/features/products/` — new feature folder, created fresh
---
```

## Core principles

- **Never truncate** existing entries.
- **Always include file paths** for created/modified files.
- **Be specific** in "Next Session Should" — vague entries are useless.
- **Keep entries focused** — ~20 lines max.
- **Commit `session-log.md`** alongside code so it tracks git history.

## What this skill does not do

- Restore context at session start — that's `session-restore` (the pair).
- Replace commit messages — it's session-level continuity, not per-change history.
- Log secrets or tokens — never write credentials into the log.

## Success criteria

Working when: a new session can be resumed from the latest entry alone —
goal, decisions, and next step are clear without re-reading the whole codebase.

## Troubleshooting

- **`session-log.md` missing:** Step 2 creates it with a header; don't hand-author.
- **Entry too long:** trim to the goal, key decisions, and next step — detail
  belongs in the code/PR, not the log.
